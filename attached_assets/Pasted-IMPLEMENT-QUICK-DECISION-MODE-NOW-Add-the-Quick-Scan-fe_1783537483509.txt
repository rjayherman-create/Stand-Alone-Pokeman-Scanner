Main goal:

“Should I put this in my cart right now?”

BUY / MAYBE / SKIP / RESEARCH MORE

1. Exact barcode / UPC / GTIN match

- scanned store price
- category rules
- markdown strength

- risk warning

FUNCTIONS TO BUILD

1. extractItemFromScan(image)

Extract:
- retailer
- product_name
- brand
- category
- subcategory
- current_store_price
- regular_price
- clearance_price
- percent_off
- UPC / GTIN
- SKU
- DPCI
- TCIN
- Costco item number
- model number
- box_condition
- sealed_status
- confidence

2. matchProductForComps(scannedItem)

Match priority:
- UPC / GTIN exact match
- ASIN if available
- DPCI / TCIN
- SKU / model number
- brand + exact product title
- fuzzy title match

Return:
- match_confidence
- match_method
- matched_title
- matched_identifier
- warning_if_low_confidence

3. buildCompSummary(scannedItem, ebayData, amazonData)

Return:
- ebay_active_median
- ebay_sold_median
- ebay_active_range
- amazon_current_price
- amazon_30_day_average
- amazon_90_day_average
- estimated_local_facebook_sale_price
- suggested_facebook_list_price
- suggested_channel
- comp_confidence
- comp_notes

Facebook Marketplace estimate rules:
- Use 70%–85% of realistic eBay sold median for common sealed items.
- For fast local sale, price below eBay active median.
- For bulky items, reduce local estimate.
- For new sealed LEGO/toys, use eBay sold median as the strongest reference.
- Amazon is reference only unless user is approved to sell on Amazon.

4. calculateProfit(scannedItem, compSummary)

For Facebook Marketplace:
- platform fee = 0
- shipping = 0
- packaging = 0
- use local sale price
- include optional tax if user enables it

For eBay:
- include estimated platform fees
- include shipping
- include packing
- calculate net profit, not gross spread

Return:
- gross_spread
- estimated_net_profit
- profit_margin_percent
- negotiation_floor
- max_buy_price
- recommended_quantity

5. makeQuickDecision(scannedItem, compSummary, profitSummary)

Return:
- recommendation: BUY / MAYBE / SKIP / RESEARCH_MORE
- confidence_score
- one_sentence_reason
- risk_warning
- max_quantity
- best_next_action

DECISION RULES

BUY:
- match confidence 75+
- estimated profit meets category minimum
- item is not bulky
- item is not visibly damaged
- demand category is strong
- local sale price gives enough spread

MAYBE:
- profit exists but comps are mixed
- confidence is medium
- item may be bulky
- buy only 1 test unit

SKIP:
- not enough spread
- eBay/Amazon price too close to store price
- damaged box
- heavy/bulky with low profit
- niche or slow-moving item

RESEARCH_MORE:
- product identity unclear
- barcode/model missing
- comps unavailable
- Amazon/eBay results conflict
- price missing
- category is unfamiliar

ERROR STATES

“eBay comps unavailable. Try scanning the barcode or entering the model number.”

“Amazon data source not connected or unavailable. Decision is based on store price and eBay/local comps.”

“Comps unavailable. Save this as Research More or enter a manual comp.”

If product identity is unclear:
“Scan the barcode, model number, or front of the box for a better match.”

“Price not detected. Enter the store price to calculate profit.”
